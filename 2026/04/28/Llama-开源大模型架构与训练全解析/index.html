Llama：开源大模型架构与训练全解析

Meta 开源的 Llama 系列模型，从 Llama 1 到 Llama 3，不仅性能追上了闭源巨头，更在架构设计和训练方法上形成了自己的一套方法论。这篇文章记录了我学习 Llama 过程中梳理的核心要点。

一、Llama 是什么

Llama（Large Language Model Meta AI）是 Meta 发布的开源稠密 Transformer 模型。

这里有两个关键词值得注意：

  • 开源：权重和架构都公开，社区可以在此基础上微调、改进，催生出了 Alpaca、Vicuna、Llama-2-Chat 等一系列衍生模型
  • 稠密模型：每次推理都会激活全部参数（与之相对的是 MoE 混合专家模型，只激活部分参数）

Llama 的核心理念是：用更好的架构设计和更充分的训练，在相对较小的参数量下达到顶尖效果。比如 Llama-2-70B 就能在很多任务上追平 GPT-3.5，而 Llama 3 405B 更是直接对标闭源最强模型。

二、架构层面的四大改进

Llama 在原始 Transformer 的基础上做了四处关键改良，每一处都是为了”更快、更强、更省”。

2.1 SwiGLU：比 ReLU 更聪明的激活函数

传统 Transformer 的 FFN 使用的是 ReLU：

$$
\text{ReLU}(x) = \max(0, x)
$$

ReLU 简单高效，但有一个硬伤：在 x=0 处不可导。这会导致梯度在零点出现突变，影响优化稳定性。而且 ReLU 会把所有负数直接截断为 0，信息损失比较严重。

Llama 换成了 SwiGLU，它的形式是：

$$
\text{SwiGLU}(x, W, V) = \text{Swish}(xW) \otimes (xV)
$$

其中 $\text{Swish}(x) = x \cdot \sigma(x)$，$\sigma$ 是 Sigmoid 函数。SwiGLU 的核心优势：

  • 处处可导：没有 ReLU 的”硬拐角”
  • 非线性更强：Sigmoid 门控机制让模型能更精细地控制信息流
  • 表达能力更强：实验表明 SwiGLU 在各种 NLP 任务上 consistently 优于 ReLU 和 GELU

2.2 RMSNorm：LayerNorm 的瘦身版

Transformer 原始论文用的是 LayerNorm：

$$
\text{LayerNorm}(x) = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \cdot \gamma
$$

需要先算均值 $\mu$，再算方差 $\sigma^2$。

Llama 改用 RMSNorm（Root Mean Square Normalization）：

$$
\text{RMSNorm}(x) = \frac{x}{\sqrt{\text{RMS}(x)^2 + \epsilon}} \cdot \gamma
$$

其中 $\text{RMS}(x) = \sqrt{\frac{1}{d} \sum_{i=1}^{d} x_i^2}$。

关键区别：RMSNorm 不做去均值操作，直接用均方根做缩放。这样做有两个好处：

  • 计算量减少：省掉了求均值的那一步
  • 加速推理和训练：尤其是在超长序列场景下，累积均值的开销不容忽视

2.3 RoPE：让位置编码”转”起来

原始的 Transformer 使用正弦/余弦位置编码，把位置向量直接加到词嵌入上。这种编码有个问题：它只编码了绝对位置，没有显式建模词与词之间的相对位置关系。

举个例子：

  • “我 喜欢 篮球” 中，”我”和”篮球”的距离是 2
  • “他 吃 了 早饭” 中，”他”和”早饭”的距离也是 2

传统编码会把这两对词处理成相同的”位置关系”，但显然它们的语义关系完全不同。

Llama 引入了 RoPE（Rotary Position Embedding，旋转位置编码）。它的核心思想是：不在词嵌入上做加法，而是对词嵌入做旋转。

具体做法是把每个 token 的嵌入向量按照其在序列中的位置，乘以一个旋转矩阵：

$$
R_m = \begin{pmatrix}
\cos(m\theta) & -\sin(m\theta) \
\sin(m\theta) & \cos(m\theta)
\end{pmatrix}
$$

第 $m$ 个 token 的嵌入会旋转 $m\theta$ 角度。通过这种方式：

  • 不同位置的 token 旋转角度不同，天然编码了位置信息
  • 点积运算时会自动涌现出相对位置信息：$q_m^T k_n$ 的结果只与 $m-n$ 有关
  • 外推性更好：模型对训练时没见过的长序列也能有不错的表现

2.4 GQA：多头注意力的”精简版”

标准的 Multi-Head Attention（MHA） 里，每个注意力头都有自己的 Q、K、V 三套投影矩阵。假设有 32 个头，那就是 32 组 K 和 32 组 V。

GQA（Grouped Query Attention） 的思路是：Q 保持多头，但 K 和 V 按组共享。

在 Llama 的实现中，通常是 4 个 Q 头共享 1 个 K 头和 1 个 V 头（即 4q-1k-1v）。打个比方：

  • MHA 像 32 个人各自去查完整的图书馆目录（32 套 K/V）
  • GQA 像 32 个人分成 8 组，每组共享一份目录（8 套 K/V）

这样做的好处：

  • 显存减少：KV Cache 的存储量从 $O(n_{head})$ 降到 $O(n_{group})$
  • 推理加速：解码时读取 KV Cache 的带宽压力大幅降低
  • 精度损失可控：实验证明分组后模型能力基本不打折

三、训练流程：从”文盲”到”专家”

Llama 的训练不是一步到位的，而是分阶段逐步打磨：

3.1 预训练：海量数据灌入

在大规模的互联网文本上做自监督学习，让模型学会语言的统计规律。Llama 3 的预训练数据量达到了 15 万亿 token，是 Llama 2 的 7 倍。

3.2 指令微调（SFT）：教模型”听话”

预训练后的模型虽然能生成流畅的文本，但它不知道该怎么”回答问题”、”执行指令”。

指令微调就是构造大量高质量的（指令，回答）问答对，像教婴儿一样告诉模型：”当你看到这样的问题时，应该这样回答”。

SFT 的数据质量比数量更重要——几千条精心编写的指令对，可能比几十万条低质量数据效果更好。

3.3 人类反馈的强化学习：让模型”懂事”

SFT 之后的模型虽然能回答问题，但回答不一定符合人类的偏好（比如可能产生有害内容、啰嗦、不够有帮助）。

RLHF（Reinforcement Learning from Human Feedback）的流程是：

  1. 训练 Reward Model：让人类标注员对模型的多个回答打分，训练一个”评分模型”
  2. 用 RL 优化策略：让模型生成回答，Reward Model 打分，模型朝着高分方向优化

这个过程有点像公司的 OKR/KPI 体系：Reward Model 就是考核标准，模型根据考核结果调整自己的行为。

不同模型的技术路线选择：

模型 对齐方法
Llama 2/3 DPO（Direct Preference Optimization）
DeepSeek PPO（Proximal Policy Optimization）

DPO vs PPO：

  • PPO 是经典的 RL 算法，需要维护 Reward Model 和策略模型，流程复杂但效果上限高
  • DPO 更简洁，直接用偏好数据优化，不需要显式的 Reward Model，训练更稳定

Llama 3 使用了六轮 DPO 训练，逐步提升模型表现——不是一次到位，而是反复迭代、精益求精。

3.4 拒绝采样：数据质量是生命线

在训练过程中，Llama 团队非常注重拒绝采样（Rejection Sampling）：

  • 从模型生成的大量样本中，筛选出质量高的保留，差的丢掉
  • 同时过滤掉有害、偏见、低质量的数据
  • 用筛选后的优质数据重新进行 SFT 或 DPO

数据质量决定了模型的天花板。同样的架构，用不同的数据训练，效果可能天差地别。

四、Llama 3 的关键参数

指标 Llama 3
最大参数量 405B
架构 稠密 Transformer
上下文窗口 128K（从 32K 扩展而来）
训练数据 15T tokens
分词器 BPE（与 GPT 相同）
位置编码 RoPE
归一化 RMSNorm
激活函数 SwiGLU
注意力 GQA

Llama 3 上下文窗口从 32K 扩展到 128K，意味着它能一次性处理大约 10 万字 的文本，相当于一本中篇小说的长度。这对于长文档总结、代码分析等任务至关重要。

五、BPE 分词器

Llama 和 GPT 系列都使用 BPE（Byte Pair Encoding） 作为分词器。BPE 的核心思想是：

  1. 从字符级别的词表开始
  2. 统计训练数据中相邻字符对的频率
  3. 把频率最高的字符对合并成一个新的 token
  4. 重复上述过程，直到词表达到目标大小

比如 “学习” 如果在语料中频繁出现，BPE 会把它合并成一个独立的 token，而不是拆成 “学” 和 “习” 两个。

BPE 的优势是平衡了词表大小和表达能力：既不会像纯字符分词那样序列过长，也不会像纯词语分词那样词表爆炸。

六、总结

Llama 的成功不是单一技术的突破，而是架构优化 + 训练方法 + 数据质量三者协同的结果：

层面 关键改进 目的
激活函数 ReLU → SwiGLU 更强的非线性，处处可导
归一化 LayerNorm → RMSNorm 减少计算，加速推理
位置编码 正弦编码 → RoPE 显式建模相对位置关系
注意力 MHA → GQA 降低 KV Cache，提升推理效率
训练方法 SFT + DPO + 拒绝采样 对齐人类偏好，保证输出质量
数据规模 15T tokens 更充分的预训练

从 Llama 1 到 Llama 3，Meta 证明了开源模型完全可以在性能上追赶甚至超越闭源对手。而对于学习者来说，Llama 公开的论文和技术细节，也是理解现代大模型最好的教材之一。
