DeepSeek R1：推理模型的强化学习之路

DeepSeek R1 可能是 2025 年最出圈的开源模型之一。它不仅以极低的训练成本追平了 OpenAI o1 的推理能力，更关键的是公开了完整的训练方法——尤其是 R1-Zero 这条”纯 RL 路线”，让人们第一次看到了：不依赖人工标注，模型自己就能涌现出推理能力。

一、R1-Zero：纯强化学习的”野生”推理

在讲 R1 之前，必须先讲它的前辈 R1-Zero。

传统的大模型训练路径是：预训练 → SFT（指令微调）→ RLHF（强化学习对齐）。R1-Zero 打破了这个惯例——它没有经过任何 SFT，直接上强化学习。

1.1 GRPO：群体相对策略优化

R1-Zero 使用的核心算法是 GRPO（Group Relative Policy Optimization），可以理解为 PPO 的”简化版”：

  • 针对同一个 Prompt，让模型生成 8/16/64 个不同的回答
  • 用奖励模型给这些回答打分
  • 模型朝着高分回答的方向优化
  • 同时用 KL 散度约束，防止模型偏离太远

GRPO 和 PPO 最大的区别是：不需要额外的 Critic 网络。PPO 需要训练一个”裁判”来评估每次生成的价值，GRPO 直接用组内相对排名来算优势函数，大大简化了训练流程。

1.2 奖励模型：只用结果，不用过程

R1-Zero 的奖励设计非常简洁——只判断最终答案对不对，不评判中间推理过程。

为什么不用”过程奖励模型”（PRM）？因为 PRM 可能会导致 奖励劫持（Reward Hacking）：模型学会讨好评分标准，而不是真正解决问题。比如评分器看重”步骤是否详细”，模型就会故意写很多废话来刷分。

结果奖励虽然粗糙，但目标明确，不容易被钻空子。

1.3 推理格式的自我进化

R1-Zero 在训练过程中自己学会了组织推理格式：

没有人教它这么写。纯粹是通过 RL 的奖励反馈，模型发现：把推理过程结构化地呈现出来，更容易得到正确答案。

1.4 “Aha Moment”：模型自己顿悟了

R1-Zero 的训练日志中出现过这样的现象：

模型在推理中途突然发现之前的思路错了，然后写下 “Wait, wait, wait…”，重新换了一种方法继续推理。

DeepSeek 团队把这个称为模型的 “Aha Moment”——就像人类解题时突然灵光一闪。这个时刻不是人为设计的，而是纯 RL 训练过程中自发涌现出来的。

1.5 R1-Zero 的缺陷

纯 RL 路线虽然惊艳，但有两个明显问题：

  • 可读性差：推理过程像草稿纸，格式混乱，人类很难跟着看
  • 语言混杂：中文问题里突然冒出英文推理，或者各种语言混搭

这些问题说明：只练”对”，不练”好看”和”规范”，模型虽然能解题，但不够实用。

二、R1：冷启动 + 四阶段训练

为了解决 R1-Zero 的问题，DeepSeek 推出了完整的 R1 模型。它的训练加入了 冷启动数据 和更精细的四阶段流程。

一个小知识：DeepSeek App 里开启”深度思考”就是 R1，不开就是 V3。

2.1 第一阶段：冷启动（Cold Start）

R1-Zero 的最大问题是”从零开始 RL”，模型没有任何”规范推理”的先验知识。

冷启动阶段的核心任务是：构造少量高质量的推理数据，先让模型”学会怎么写”。

具体做法：

  • Few-shot 提示：给模型几个带详细推理过程的示例，让它照着格式生成
  • R1-Zero 输出清洗：把 R1-Zero 生成的推理结果做可读化处理（格式化、去混杂语言）
  • 人工后期标注：对关键样本进行人工精修

这批数据量不大，但质量很高。用它做一轮 SFT，模型就具备了”像人一样一步步推理”的基本能力。

2.2 第二阶段：面向推理的强化学习

冷启动后的模型已经会”规范推理”了，接下来用 GRPO 继续训练，但这次聚焦在推理类任务上（数学、代码、逻辑题等）。

这一阶段主要优化 Chain-of-Thought（CoT，思维链） 能力：

  • 让模型生成详细的中间推理步骤
  • 奖励模型评估最终答案的正确性
  • 模型学会”想得更深、更细”

2.3 第三阶段：拒绝采样与全面 SFT

第二阶段 RL 训练后的模型已经很强了，DeepSeek 用它来做数据生成器：

  1. 用这个强模型生成大量回答
  2. 拒绝采样：只保留答案正确、推理过程清晰、格式规范的数据
  3. 用筛选后的优质数据做全面的 SFT

这一轮 SFT 不局限于推理任务，还包括通用对话、写作、知识问答等，让模型在保持推理能力的同时，通用能力也不掉队。

2.4 第四阶段：面向全场景的强化学习

最后再来一轮 RL，但这次覆盖所有场景：

  • 推理任务：继续用结果奖励优化
  • 通用任务：用人类偏好数据优化（有用性、无害性、诚实性）
  • 最终得到一个在推理能力和通用能力上都很强的模型

三、训练路线对比：R1-Zero vs R1 vs 传统路线

维度 R1-Zero R1（完整版） 传统 GPT 路线
SFT ❌ 完全没有 ✅ 冷启动 + 全面 SFT ✅ 大量人工 SFT
RL ✅ GRPO 纯 RL ✅ 多轮 RL ✅ PPO/RLHF
数据依赖 极低（只需要答案对错） 中等（需要少量冷启动数据） 很高（需要大量偏好标注）
推理能力 强，但粗糙 强且规范 依赖规模
可读性 差 好 好
语言混杂 严重 解决 无

R1-Zero 的意义在于证明了：推理能力可以通过纯 RL 自发涌现，不需要昂贵的人工标注。这为后续研究指明了一个方向——让模型自己教自己。

四、知识蒸馏：让大模型带小模型

R1 训练完成后，DeepSeek 还做了一件事：知识蒸馏。

核心思路很简单：让 R1（大模型）生成大量高质量的推理数据，然后用这些数据去微调更小的模型（如 Qwen、Llama 的 7B/14B 版本）。

效果惊人：

  • 蒸馏后的 14B 小模型，推理能力超过了原始的 32B 模型
  • 这说明数据质量比模型规模更重要
  • 也为”小模型 + 优质数据”这条路线提供了有力证据

蒸馏的本质是知识迁移——大模型学到的推理模式、解题策略，通过数据传递给小模型，让小模型”站在巨人肩膀上”。

五、DeepSeek 的启示

R1 的论文给业界带来了几个重要启示：

5.1 RL 可以替代大量人工标注

R1-Zero 证明了：只要奖励信号设计得当，模型自己就能探索出复杂的推理策略。这意味着未来训练推理模型时，人工标注的需求可能大幅下降。

5.2 冷启动数据不需要很多

R1 的冷启动阶段只用了少量高质量数据，远少于传统 SFT 的规模。这说明数据质量比数量重要得多——几百条精心设计的示例，可能比几万条低质量数据更有效。

5.3 推理能力可以”自发涌现”

“Aha Moment” 是最震撼的发现。模型在 RL 训练中自己学会了反思、纠错、换思路——这些能力不是人类教的，是奖励信号引导下的自然涌现。

5.4 开源的力量

DeepSeek 不仅开源了模型权重，还开源了训练方法。这让整个社区都能基于 R1 做研究、改进、应用。从 R1 衍生出的各种蒸馏模型、微调版本，正在快速推动推理能力的平民化。

六、总结

DeepSeek R1 的训练是一条从”野蛮生长”到”精修打磨”的完整路径：

这条路线的核心洞见是：先让模型自己探索出能力（RL），再用少量高质量数据教会它规范表达（冷启动），最后用迭代优化打磨细节。

对于学习者来说，R1 的论文最大的价值不是某个具体的技术点，而是它展示了一种可能性——当我们把奖励信号设计好，模型可以比我们想象的更聪明。

七、从 R1 到 V4：架构的全面升级

R1 之后，DeepSeek 在 2026 年 4 月 24 日发布了 DeepSeek V4，这不仅是参数的扩展，更是一次架构级的重新设计。

7.1 稠密 → MoE：参数多，激活少

R1 和 V3 都是稠密模型——每次推理都要跑完全部参数。V4 换成了 MoE（Mixture of Experts，混合专家） 架构：

版本 总参数量 每 token 激活参数 MoE 配置
V4-Pro 1.6T 49B 1 个共享专家 + 384 个路由专家，每 token 激活 6 个
V4-Flash 284B 13B 1 个共享专家 + 256 个路由专家，每 token 激活 6 个

打个比方：模型像一个由几百位专家组成的顾问团，每次遇到问题只请其中 6 位最相关的专家来回答，既保证了能力，又大幅降低了推理成本。

7.2 上下文窗口暴增至 1M

V3 的上下文窗口是 128K，V4 直接拉到 100 万 token——相当于能一次性读完一本中篇小说。

支撑这个数字的是新的注意力机制 CSA + HCA（双重压缩注意力）：

  • 长文本计算量降低 73%
  • KV Cache 显存占用降至 V3 的 10%
  • 这意味着以前跑不动的长文档分析，现在成本降到了可接受的范围

7.3 训练方法再次进化

V4 的后训练阶段完全抛弃了传统的 RLHF，改用 On-Policy Distillation (OPD)：

  • 先在数学、代码、Agent、指令遵循等领域分别训练专家模型
  • 再把多个领域的专家能力合并到同一个模型中
  • 相比 RLHF，OPD 训练更稳定，不同能力之间的冲突更小

预训练数据也翻倍到了 33T tokens（V3 是 14.8T），上下文长度分四阶段逐步扩展（4K → 16K → 64K → 1M）。

7.4 深度拥抱国产算力

V4 最值得关注的一点是国产芯片的深度适配：

  • 华为昇腾 950 超节点：全面支持训练和推理，芯模协同优化
  • 寒武纪、摩尔线程：发布当日完成 Day-0 适配，代码已开源
  • 官方技术报告首次把国产芯片和英伟达 GPU 写入同一份硬件验证清单

这释放了一个明确信号：DeepSeek 正在构建一条不依赖英伟达的技术路线。

7.5 定价与可用性

V4 提供两个版本：

  • V4-Flash：轻量高速，输出约 ¥2/百万 tokens，适合日常对话和通用任务
  • V4-Pro：旗舰性能，限时优惠期间缓存命中输入低至 ¥0.25/百万 tokens

旧版 deepseek-chat 和 deepseek-reasoner 接口将在 2026 年 7 月 24 日退役，目前已自动路由到 V4-Flash 的非思考模式和思考模式。

7.6 一条持续演进的技术路线

从 R1-Zero 到 R1 再到 V4，DeepSeek 展示了一条清晰的发展路径：

每一步都不是孤立的技术点，而是围绕”如何用更少的成本、更开放的方式，做出更强的模型“这个核心目标持续迭代。
